# Catalog Restructure + Traefik Removal Design

**Date:** 2026-04-14
**Branch:** feat/catalog-restructure

## Overview

Two-phase restructure of the Stackr catalog and core engine:

- **Phase 1** — Catalog cleanup: remove 13 apps, add 3 new NPM-native apps (seerr, pocket-id, tinyauth), update tests and CI.
- **Phase 2** — Remove Traefik from the engine: delete `TraefikConfig`, `traefik_labels()`, `middleware.py`, `dns_providers.py`, strip labels from all remaining templates, and remove all Traefik-related tests.

NPM (nginx-proxy-manager) becomes the sole reverse proxy. Apps remain on the `proxy` Docker network so NPM can route to them by container name.

### Apps Added (3)

All new apps are designed NPM-native: they have no `labels:` block, no `traefik_labels()` call, and no `requires: [traefik]` in their `app.yml`.

#### `media/seerr`

- **Image:** `seerr/seerr:latest`
- **Port:** 5055
- **Storage:** single config volume at `/app/config`
- **Health check (CI):** `container_port: 5055, health_path: "/"`
- Replaces both overseerr and jellyseerr. Supports Plex, Jellyfin, and Emby.
- No sidecar required (SQLite default).

#### `security/pocket-id`

- **Image:** `stonith404/pocket-id:latest`
- **Port:** 1411
- **Storage:** data volume at `/app/data` (SQLite, no DB sidecar)
- **Health check (CI):** `container_port: 1411, health_path: "/"`
- OIDC provider with passkey-only authentication.
- Exposes OIDC endpoints for apps with native OIDC support (e.g., Grafana, Gitea, Portainer).
- `PUBLIC_APP_URL` is set from `network.domain` so OIDC redirects resolve correctly.
- `suggests: [tinyauth]` — apps without native OIDC can be protected via tinyauth.

#### `security/tinyauth`

- **Image:** `ghcr.io/steveiliop56/tinyauth:latest`
- **Port:** 3000
- **Storage:** none (stateless)
- **Health check (CI):** `container_port: 3000, health_path: "/"`
- Lightweight forward-auth proxy; sits between NPM and apps that lack native OIDC.
- Configured via env vars to use Pocket ID as the OIDC backend:
  - `SECRET` → `${TINYAUTH_SECRET}` (random; auto-generated by stackr)
  - `OAUTH_CLIENT_ID` → `${TINYAUTH_OAUTH_CLIENT_ID}`
  - `OAUTH_CLIENT_SECRET` → `${TINYAUTH_OAUTH_CLIENT_SECRET}`
  - Auth/token/userinfo URLs point to the Pocket ID container by service name.
- `requires: [pocket-id]` — must be deployed after pocket-id.

## Phase 2: Remove Traefik from the Engine

### `stackr/config.py`

- Delete `TraefikConfig` class entirely.
- Remove `traefik: TraefikConfig` field from `StackrConfig`.
- Simplify `NetworkConfig`: remove `mode` (external/internal/hybrid was only meaningful for Traefik cert resolvers). Keep `domain` and `local_domain`.
- Update `stackr.yml.example`: remove the `traefik:` section and `network.mode`.

### `stackr/renderer.py`

- Delete `traefik_labels()` function.
- Remove `traefik` and `network.mode` from the Jinja2 template context.
- Remaining context variables: `global`, `network` (domain + local_domain), `security`, `vars`, `app`.

### `stackr/middleware.py`

- Delete the entire file. It exists solely to generate Traefik forward-auth labels.

### `stackr/dns_providers.py`

- Delete the entire file. It exists solely to validate Traefik DNS-01 ACME challenge providers.

### `stackr/validator.py`

- Remove `_check_traefik_config()`.
- Remove `_check_dns_provider()`.
- Remove `_check_security_stack()` — this validates `auth_provider` against authentik/authelia, both of which are removed. The `security.auth_provider` field in config can be removed or simplified.
- Update module docstring to remove the deleted checks.
- Keep: `_check_secrets()`, `_check_port_conflicts()`, `_check_deps()`, `_check_volumes()`.

### `stackr/doctor.py`

- Remove the Traefik reachability check.
- Keep all other pre-flight checks (Docker, networks, secrets, catalog).

### `stackr/tui.py`

- Remove the Traefik config display section from the settings panel.

### `stackr/web/routes.py`

- Remove the `traefik` section from the settings API (`GET /api/settings`, `POST /api/settings`).
- Remove `_save_all_settings()` Traefik key handling.

### `stackr/cli.py`

- Remove any Traefik-specific references (config display, init prompts for Traefik).

### `stackr/migrate.py`

- Remove Traefik-related entries from `_DEPLOYRR_MAP`.

### All 37 Remaining Compose Templates

For every `catalog/<category>/<app>/compose.yml.j2`:
- Remove the `labels:` block and the `{% for k, v in traefik_labels(port).items() %}` loop.
- Keep the `proxy:` network stanza (NPM still needs to reach containers on this network).

For every `catalog/<category>/<app>/app.yml`:
- Remove `requires: [traefik]` (or the `traefik` entry from the requires list).
- Remove `exposure` field — without Traefik cert resolvers, external/internal/hybrid has no effect on rendering.

## Network Architecture After Both Phases

Apps continue to join the `proxy` Docker network (created once by `stackr init` or manually). NPM is also on this network and routes to apps by container name and port. No labels, no automatic discovery — routing is configured through NPM's web UI.

The `socket_proxy` network and the `socket-proxy` app remain for users who want Docker socket isolation (portainer, dozzle, watchtower still benefit from it).
